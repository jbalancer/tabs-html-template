Main tag

<div class="TAB_TYPE TAB_STYLE">
</div>

This is the parent tag of the whole tab and it is important that it has 2 classes defined - that is, instead of TAB_TYPE, you need to write one of the following 4 available classes:

  1. new-tab-wrapper - This is the usual tab, in which the elements of switching tabs are on the left in the column, and on mobile devices they are located exactly in the column;
  2. icon-tab-wrapper - This is a tab with icons, in which the controls act as icons and are arranged in a row, and on mobile devices they are located exactly in the column;
  3. mob-tab-wrapper - it does not matter - the user uses the phone or computer, this is a universal type of taba and can be used everywhere Here, the controls are initially located exactly in the column, and the icons along with the text appear;
  4. button-tab-wrapper - This is a tab in which the controls are buttons separated from the content and located on the left, and on mobile devices they are located exactly in the column.

Tabs

<div class="tab-content">
</div>

This is the parent tag of the entire tab with the tab-content class and the number of tabs depends on its number.

  • This element must be inside the parent tag of the whole tab and be the parent of the other tags;
  • For example, there are only 5 tabs in the current tab and each tab is 1 tag with the class tab-content.

The discoverers

<input type="radio" name="SAME_NAME" class="content-discoverer" id="SAME_TEXT">

This is a tag with the content-discoverer class, which must be inside the element with the tab-content class and be its first child. It is hidden and is a control when selected. It is important that this element has the following 4 attributes:

  • type - if radio, then when selecting a tab, the rest will be hidden. If checkbox, then when you select the tab, the rest will remain active. CHECKBOX recommended only for tabs with class mob-tab-wrapper;
  • name - this attribute must contain the same NAME in all deposits of the current tab;
  • id - this attribute must contain a unique identifier and contain the same value with the attribute for the right element label;
  • class - value is content-discoverer.

Titles and menu icons

<label class="tab-titles" for="SAME_TEXT">
<h3>Title …</h3>
</label>

The tag in the label menu should have an for attribute whose value should be the id attribute value of input located on the left for activity tabs when selecting the menu title. Also label should contain 2 tags:

  1. h3(*) - title tag not necessary h3, though recommended. It must contain the NAME or TITLE tab, also the tag i:
    • if tab type - icon-tab-wrapper, then the title should contain only an icon;
    • if tab type - mob-tab-wrapper, then the title should contain not only the icon, but also the name or the title.
  2. i - it is necessary that i should act as a tag, it should also be an icon. (a free service providing more than 1500 free icons fontawesome.com). For example, in the current tab, the icon is the following element:
<i class="fab fa-html5"></i>

Description

<div class="tab-descriptions">Description …</div>
</div>

Description or content of the tab, no difference - here you can arrange the contents of the tab with your design! It must be inside the tab tag with the tab-content class and after the label tag with the tab-titles class.

  • div(*) - recommended but not necessary div;
  • tab-descriptions - required class for this element.

All CSS files

You do not need to use all the style files, only 2 files are required - "style.css" and "media.css", or a compressed version that contains all styles "tabs-min.css", or "tabs.css", which combines all css-files.

  • "tabs-min.css" - All styles are fully compressed. Path - "build/ready/assets/css/compressed/tabs-min.css";
  • "tabs.css" - combining all styles, without compression. Path - "build/ready/assets/css/compressed/tabs.css";
  • "style.css" - main styles (with large screen sizes). Path - "build/ready/assets/css/main/style.css";
  • "media.css" - main styles (with small screen sizes). Path - "build/ready/assets/css/main/media.css";
  • "alice-blue.css" - color scheme styles (light blue gradient). Path - "build/ready/assets/css/theme/alice-blue.css";
  • "dark-red.css" - color scheme styles (dark red gradient). Path - "build/ready/assets/css/theme/dark-red.css";
  • "navy-blue.css" - color scheme styles (dark blue gradient). Path - "build/ready/assets/css/theme/navy-blue.css";
  • "without-bg.css" - color scheme styles (light blue only for frames and shadows). Path - "build/ready/assets/css/theme/without-bg.css";
  • "new-bg.css" - color scheme styles for editing. Path - "build/new/css/theme/new-bg.css";
  • "description.css" - styles for descriptions inside taba. Path - "build/ready/assets/css/main/description.css".

Ready styles

The CSS files are pre-connected to the page and all files contain comments describing the CSS rules.

  1. If you want to use all types of tabs and all styles for them, then you can include only a single file-style. "tabs-min.css";
  2. style.css and "media.css', must be connected if not connected tabs-min.css, they are needed to display tabs according to their type;
  3. If you need only a few or one finished color scheme of the taba, then you can connect the styles you need separately:
    • "alice-blue.css";
    • "dark-red.css";
    • "navy-blue.css";
    • "without-bg.css".
  4. Further, if you wish, you can connect ready-made styles for descriptions. "description.css".

New design

It is recommended to edit only the file (new-bg.css) in the theme folder, where examples of properties are recommended to be changed. The value of the class of editable tabs is new-bg. Files that are connected by default:

  • "style.css" - This file contains styles for all tabs, with large screens;
  • "media.css" - This file contains styles for all tabs, with small screens;
  • "description.css" - the default description file, you can use it if you wish, and if you want it, the HTML code of your tag with the tab-descriptions class will look like this:
<div class="tab-descriptions">
<div class="image"></div>
<div class="text">
<h1>Title …</h1>
<p>Text …</p>
</div>
</div>
  • For the image class, the images are inserted using the :before pseudo-element, so you must change the path to the image in the description.css file, starting at line 32. This is done to ensure that the same picture is adaptive in different screen sizes;
  • An element with the text class should contain only 1 h1 tag, which should be placed first;
  • In the tags p you can place the text itself.

Use of pre-made themes

The following steps will help you quickly integrate tabs into your web project:

  1. Open the folder ready - "build/ready";
  2. Select a tab topic in the folder theme - "build/ready/theme;
  3. Next, select the type of tobacco, one of the following files:
    • "button-tab.html"
    • "icon-tab.html"
    • "mobile-tab.html"
    • "simple-tab.html"
  4. Now, after you have chosen the tab you like, you have 2 ways:
    • Select the type of tab right here and the HTML sources for 5 tabs will be available on top, you will also see a design;
    • You can copy the source code in the body tag into your web project (for professionals).
  5. Now, after you have inserted the HTML sources into your web project, you have 2 ways again:
    • Or, add the following 3 files to your web project in the head tag, which are in the folder "css":
      • "style.css" - build/ready/assets/css/main/style.css;
      • "media.css" - build/ready/assets/css/main/media.css;
      • what you chose - build/ready/assets/css/theme/.

      This is what should happen:

      <head>
      </link rel="stylesheet" href="/style.css">
      </link rel="stylesheet" href="/media.css">
      </link rel="stylesheet" href="/THEME.css">
      </head>
    • Or copy the source code above and include the file tabs-min.css in your web project - "build/ready/assets/css/compressed/tabs-min.css":
      <head>
      </link rel="stylesheet" href="/tabs-min.css">
      </head>

New design

  1. Open the folder new - "build/new";
  2. Select a file with the required type of tab, that is, one of the following four types of tabs:
    • "button-tab.html" - "build/new/button-tab.html"
    • "icon-tab.html" - "build/new/icon-tab.html"
    • "mobile-tab.html" - "build/new/mobile-tab.html"
    • "simple-tab.html" - "build/new/simple-tab.html"
  3. By default, tabs will have style - without-bg and to change the design, you need to edit the file new-bg.css - "build/new/css/theme/new-bg.css", where styles are given as an example and it is recommended to use exactly those styles;
  4. Then you can either connect "style.css", "main.css" and "new-bg.css", either merge "new-bg.css" with "tabs.css" and connect only one file, while ready-made styles will also be available to you.
